Prerequisitos — solo dos cosas
Git for Windows + Go 1.24 · No necesitas nada más
⊞
Requisitos reales para Windows 11

Claude Code: solo Git for Windows (usa Git Bash internamente). No requiere Node.js, no requiere WSL2.

Gentle-AI: binario de Go autocontenido. No requiere Node.js, no requiere Python, no requiere Docker.

Go 1.24+: solo si usas go install para instalar gentle-ai. Con el installer PowerShell oficial no lo necesitas.

Paso 1 — Git for Windows (prerequisito de Claude Code)

PowerShell 7 — como Administrador
# Opción A — winget (recomendado)
winget install --id Git.Git -e --source winget

# Opción B — Descargar instalador gráfico desde git-scm.com
# Durante la instalación: marcar "Add Git to PATH" (ya viene marcado por defecto)

# Verificar instalación (cerrar y reabrir PowerShell primero)
git --version     # → git version 2.x.x.windows.x
git --exec-path   # debe mostrar ruta de Git Bash

Paso 2 — Go 1.24 (solo si usas go install para gentle-ai)

PowerShell 7 — opcional, solo si prefieres go install
# Instalar Go via winget
winget install --id GoLang.Go -e

# Cerrar y reabrir PowerShell, luego verificar
go version   # → go version go1.24.x windows/amd64

# Agregar el bin de Go al PATH (si no lo hace automáticamente)
# Panel de Control → Variables de Entorno → PATH → agregar:
# C:\Users\TuUsuario\go\bin
✅
Si usas el installer PowerShell de gentle-ai — no necesitas Go

El script install.ps1 descarga directamente el binario compilado desde GitHub Releases. Ni Go ni Node.js requeridos. Es la opción más simple para Windows.



1
Instalar Claude Code en Windows 11
Installer oficial de Anthropic — nativo, sin dependencias, auto-actualizable

Instalación oficial (recomendada)

PowerShell 7 — NO necesitas ejecutar como Administrador
# Installer oficial de Anthropic — descarga y configura el PATH automáticamente
irm https://claude.ai/install.ps1 | iex

# Cerrar y reabrir PowerShell (necesario para que el PATH tome efecto)

# Verificar instalación
claude --version

# Diagnosticar si algo falla
claude doctor

Instalación alternativa via winget

PowerShell 7
# Instalar via Windows Package Manager
winget install --id Anthropic.ClaudeCode -e

# NOTA: winget NO auto-actualiza — tienes que correr esto manualmente:
winget upgrade Anthropic.ClaudeCode
ℹ️
El installer nativo vs npm — usa siempre el nativo

El método antiguo era npm install -g @anthropic-ai/claude-code. Ese método está oficialmente deprecado. El installer nativo (irm ... | iex) es más rápido, no requiere Node.js y se auto-actualiza en background.

Dónde instala Claude Code en Windows

# El binario se instala en:
C:\Users\TuUsuario\.local\bin\claude.exe

# La configuración queda en:
C:\Users\TuUsuario\.claude\

# Si claude no se reconoce: agregar el bin al PATH manualmente
[Environment]::SetEnvironmentVariable(
    "PATH",
    "$env:PATH;$env:USERPROFILE\.local\bin",
    "User"
)

Autenticación con tu cuenta Anthropic

PowerShell / CMD / Git Bash — cualquiera funciona
# Iniciar Claude Code — abre el browser para autenticarse via OAuth
claude

# El browser de Windows abre console.anthropic.com automáticamente
# Inicia sesión con tu cuenta → autoriza → vuelve al terminal
# La sesión se guarda en: C:\Users\TuUsuario\.claude\session.json
# No necesitas re-autenticarte en cada sesión

# Alternativa — autenticación via API key (para CI/CD o uso headless):
$env:ANTHROPIC_API_KEY = "sk-ant-TU-KEY-AQUI"

Terminales compatibles en Windows

✅
Claude Code funciona en los tres terminales de Windows

PowerShell 7 — recomendado para el día a día en Windows.

CMD (Command Prompt) — funciona, menos features.

Git Bash — funciona, sintaxis más cercana a Linux. Claude Code lo usa internamente.

Todos abren correctamente desde Windows Terminal (recomendado instalar desde la Microsoft Store).

Instalar Gentle-AI en Windows 11
Binario nativo de Go · Installer PowerShell oficial · Sin dependencias

Opción A — Installer PowerShell (recomendada, sin Go)

PowerShell 7
# Descarga el binario compilado directamente desde GitHub Releases
irm https://raw.githubusercontent.com/Gentleman-Programming/gentle-ai/main/scripts/install.ps1 | iex

# Verificar
gentle-ai --version

Opción B — Go install (si ya tienes Go 1.24+)

PowerShell 7
# Compilar e instalar desde el código fuente (siempre te da el latest)
go install github.com/gentleman-programming/gentle-ai/cmd/gentle-ai@latest

# El binario queda en: C:\Users\TuUsuario\go\bin\gentle-ai.exe
# Verificar
gentle-ai --version
✅
gentle-ai es un binario estático — cero dependencias en runtime

Una vez instalado, el binario funciona solo. Sin Node.js, sin Python, sin Docker, sin ningún runtime adicional. Solo el ejecutable gentle-ai.exe.

PowerShell 7

PowerShell 7

gentle-ai install --dry-run `
  --agent claude-code `
  --preset full-gentleman

# Si el plan se ve bien, ejecutar sin --dry-run:
gentle-ai install `
  --agent claude-code `
  --preset full-gentleman

# Nota: en PowerShell el backtick ` es el caracter de continuación de línea
# (equivalente al \ en bash)

Instalación por componentes individuales

PowerShell 7
# Solo los componentes que quieres (sin GGA ni persona si prefieres)
gentle-ai install `
  --agent claude-code `
  --component engram,sdd,skills,context7,permissions `
  --persona gentleman

# Agregar skills específicos después
gentle-ai install `
  --agent claude-code `
  --component skills `
  --skill react-19,typescript,go-testing

Usar en un proyecto NUEVO
Desde cero en Windows 11 · Dónde guardar los proyectos · Cómo arrancar
📁
¿Dónde guardar los proyectos en Windows?

Guarda tus proyectos en C:\Users\TuUsuario\proyectos\ o en C:\dev\. Evita paths con espacios o caracteres especiales. Claude Code funciona con paths de Windows normales desde PowerShell.

Flujo completo — proyecto nuevo desde PowerShell

═══════════════════════════════════════════════════
  PROYECTO NUEVO — paso a paso en Windows 11
═══════════════════════════════════════════════════

# 1. Crear el directorio y entrar
mkdir C:\dev\mi-app
cd C:\dev\mi-app

# 2. Inicializar git (necesario para Engram y GGA)
git init
git commit --allow-empty -m "chore: init"

# 3. (Opcional) Inicializar tu stack — ejemplo con Next.js 15
npx create-next-app@latest . --typescript --tailwind --eslint

# 4. Configurar el ecosistema Gentleman
gentle-ai install --agent claude-code --preset full-gentleman

# 5. Inicializar Engram en el proyecto
engram init

# 6. Iniciar Claude Code en el proyecto
claude

# 7. En el chat de Claude Code, escribir:
# "Inicializa el contexto SDD para este proyecto Next.js 15 con TypeScript"

Primera sesión — qué decirle a Claude

Dentro del chat de Claude Code
# Mensaje de apertura recomendado:
"Eres mi par de programación Gentleman. Estoy iniciando
un proyecto de [descripción]. Usemos SDD: primero
explorá el stack y definí la arquitectura."

# Para activar SDD en un feature:
"Usemos SDD para implementar [feature]. Empezá por la fase explore."

# Para verificar que Context7 funciona:
"¿Cuáles son los cambios de API en React 19? Consultá Context7."

# Para guardar algo en Engram:
"Guardá en Engram: usamos Zod para validación de todos los inputs."

🛡️
gentle-ai no destruye tu configuración existente

Usa marker-based file merging: inyecta los bloques del ecosistema sin borrar tu configuración. Además hace un backup automático antes de cualquier cambio. Si algo sale mal: revisa C:\Users\TuUsuario\.gentle-ai\backups\

Windows nativo vs WSL2 — ¿cuándo usar cada uno?
La única diferencia real que importa
⊞ Windows Nativo (esta guía)
🐧 WSL2 + Ubuntu

✅  Setup simple · 2 pasos

✅  Claude Code · funciona igual

✅  Gentle-AI · funciona igual

✅  Engram, SDD, Context7, GGA · todos funcionan

✅  VS Code · integración directa sin extensiones

❌  Bash sandbox de seguridad (solo WSL2)

→  Recomendado para uso personal y desarrollo

⚠  Setup más complejo · múltiples pasos

✅  Claude Code · funciona igual

✅  Gentle-AI · funciona igual

✅  Engram, SDD, Context7, GGA · todos funcionan

✅  NemoClaw completo (sandbox de seguridad NVIDIA)

✅  Bash sandbox de seguridad de Claude Code

→  Solo necesario si usas NemoClaw o OpenClaw

💡
Conclusión clara

Para Claude Code + gentle-ai en uso personal: Windows nativo es suficiente y mucho más simple. Las únicas razones para usar WSL2 son si ya lo tienes configurado, si quieres el bash sandbox de seguridad de Claude Code, o si usas OpenClaw/NemoClaw que sí requieren Linux.



⚠
Troubleshooting en Windows 11
Problemas comunes y sus soluciones
ProblemaCausaSolución
claude no se reconoce como comando ~\.local\bin no está en el PATH Cerrar y reabrir PowerShell. Si persiste:
[Environment]::SetEnvironmentVariable("PATH","$env:PATH;$env:USERPROFILE\.local\bin","User")
gentle-ai no se reconoce El binario no está en el PATH Vía go install: agregar C:\Users\TuUsuario\go\bin al PATH. Vía installer ps1: cerrar y reabrir PowerShell.
El browser no abre al autenticar Firewall o configuración de browser Copiar la URL que aparece en el terminal y pegarla manualmente en el browser.
Pegar imágenes con Ctrl+V no funciona Limitación conocida en Windows nativo Usar Alt+V para pegar imágenes del clipboard. O guardar la imagen como archivo y arrastrarla al terminal.
Engram no recuerda el proyecto El directorio no tiene git init git init → git add . → git commit -m "init" → engram init
Error de ejecución de scripts de PowerShell Política de ejecución restrictiva Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
Claude Code pierde contexto en sesiones largas Context window lleno Usar /clear y pedir: "Revisá Engram y resumí dónde estamos". Engram garantiza continuidad.
gentle-ai sobreescribe mi CLAUDE.md Conflicto de merge Usar --dry-run primero. Restaurar desde C:\Users\TuUsuario\.gentle-ai\backups\
Context7 no responde MCP server no configurado gentle-ai install --agent claude-code --component context7
GGA no hace review al commitear Git hook no instalado gentle-ai install --agent claude-code --component gga
